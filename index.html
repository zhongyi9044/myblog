操作i系统的概念功能目标

操作系统概念：是指控制和管理整个计算机系统硬件和软件资源，并且合理的组织调度计算机的工作和资源分配，以提供给用户和其它软件方便的接口和环境。他是计算机系统中最基本的系统软件。(Windows任务管理器是最直观的例子)

操作系统作为资源管理者的功能：

  1. 处理机管理
  2. 存储器管理
  3. 文件管理
  4. 设备管理

打开QQ和别人视频的过程案例：

  1. 找到QQ安装的位置–文件管理
  2. 打开QQ.exe–存储器管理
  3. QQ程序正常运行–处理机管理
  4. 和朋友视频聊天，分配了摄像头–设备管理

操作系统作为用户和硬件之间的接口的功能：

  1. 联机命令接口(用户说一句，系统做一局)
  2. 脱机命令接口(用户说一堆，系统做一堆)
  3. 程序接口(允许用户通过程序间接使用)(由一组程序接口，即一组系统调用组成)

联机+脱机=命令接口(允许用户直接使用)

类比：操作系统就像是工人，硬件就像是锤子，锯子，钉子

操作系统的特征

  1. 并发(一件事情宏观上同时发生，微观上交替发生)
  2. 共享
    互斥共享：一个时间段只允许一个进程访问该资源
    同时共享：一个时间段宏观上允许同时进行访问资源，微观上是交替访问该资源
  3. 虚拟(物理上的实体变成若干个逻辑上的对应物)
    例子：打开了5个软件，在用户看起来是5个cpu同时为自己服务，其实是一个cpu交替处理各个服务进程
  4. 异步(允许多道程序并发执行，但由于资源有限，进程的执行不是一贯到底，而是走走停停)

操作系统的发展分类

  1. 手工操作
  2. 单道批处理(引入脱机输入输出技术，并监督程序负责控制作业输入输出)
  3. 多道批处理(多道程序并发执行)

分类：

  1. 分时操作系统
  2. 实时操作系统

操作系统的运行机制与体系结构

两种指令：

  1. 特权指令
  2. 非特权指令

两种处理器状态：

  1. 核心态
  2. 用户态

两种程序：

  1. 内核程序
  2. 应用程序

特权指令，核心态，内核程序不是普通用户可以执行的
非特权指令，用户态，应用程序是普通用户可以执行的

操作系统分层：

  1. 非内核功能
  2. 内核

内核：

  1. 进程管理，存储器管理，设备管理等
  2. 时钟管理，中断处理，原语(设备驱动，cpu切换等)

包含1和2功能叫做大内核
包含2功能叫做微内核

中断和异常

中断：

  1. 中断发生时，cpu立即变成核心态
  2. 中断发生后，当前运行的进程暂停，并由操作系统内核对中断进行处理
  3. 对于不同中断信号，有不同的处理

用户态->核心态只有中断方式实现
核心态->用户态只需要执行特权指令

中断分为内中断(指令终端，硬件故障，软件中断)和外中断(外设请求，人为干预)

系统调用

应用进程通过系统调用请求操作系统服务。由操作系统代为完成操作，更安全稳定。

按功能分类：

  1. 设备管理
  2. 文件管理
  3. 进程控制
  4. 进程通信
  5. 内存管理

系统调用的过程：

  1. 传递系统调用的参数
  2. 执行陷入指令(用户态)
  3. 执行系统调用相应的服务程序(核心态)
  4. 返回用户程序

进程的定义组成组织方式

程序：一个指令序列

进程：程序段，数据段，pcb组成了进程实体。创建进程就是创建pcb。撤销进程就是撤销pcb。pcb是进程存在的唯一标志

pcb的组成：

  1. 进程描述信息
  2. 进程控制和管理信息
  3. 资源分配清单
  4. 处理机相关信息

进程的组织方式：

  1. 链接方式：按照进程状态把pcb分为多个队列，操作系统持有各个队列指针
  2. 索引方式：根据进程状态建立索引表，操作系统拥有指向各个所以表的指针

进程的特征：

  1. 动态性
  2. 并发性
  3. 独立性
  4. 异步性
  5. 结构性

进程的状态与转换

三种基本状态：

  1. 运行态：占有cpu，并且在cpu上运行
  2. 就绪态：具备运行条件，由于没有空闲cpu，暂时不能运行
  3. 阻塞态：因等待某些事件而暂时不能运行

另外两种状态：

  1. 创建态：进程正在创建，操作系统分配资源，初始化pcb
  2. 终止态：进程从系统撤销，操作系统回收资源，撤销pcb

进程同步和进程互斥

同步亦称为直接制约关系，它是指为完成某种任务而建立多个进程，这些进程需要在某些位置上协调他们的工作次序而产生制约关系。如何解决异步问题，是进程同步讨论的内容

互斥亦称简介制约关系。进程互斥指当一个进程访问某临界资源时，另一个选购要访问临界资源的进程必须等待。当访问临界资源的进程访问结束，释放资源以后，另一个进程才能访问临界资源

对临界资源的互斥访问逻辑上分为四个部分：

  1. entry section进入区
  2. critical section临界区
  3. exit section退出区
  4. remainde sectionr剩余区

互斥访问遵循原则：

  1. 空闲让进
  2. 忙则等待
  3. 有限等待
  4. 让权等待

层叠性

相同的选择器设置相同的样式，此时样式就会被覆盖，原则是就近原则，哪个样式离结构近就执行哪个样式，样式不冲突就不会层叠。

定位

使用场景：

  1. 某个元素可以自由地在盒子内移动，并且压住其他盒子
  2. 滑动窗口的时候，盒子固定

定位组成：定位模式+边偏移

定位模式：

  1. static 静态定位
  2. relative 相对定位
  3. absolute 绝对定位
  4. fixed 固定定位

边偏移：

  1. top 距离顶部
  2. bottom 距离底部
  3. left 距离左部
  4. right 距离右部

static静态定位：默认的定位方式，无定位的意思。

relative相对定位：元素移动的时候，是相对于他的位置来说的，并且如果是标准流相对定位移动的盒子会保留原来的位置，即成为脱标(脱离标准流)
格式：position：relative；

absolute绝对定位：如果没有祖先元素，或者祖先元素没有定位，那么以浏览器为准定位。如果祖先元素有定位，则以最近的有定位的祖先元素为参考点。绝对定位不在占有远来的位置，即不脱标
格式：position：absolute；

子绝父相：子元素用绝对定位，父元素用相对定位。父盒子占用位置，子盒子不占用位置随便跑。

fixed固定定位：以浏览器可视窗口作为参照移动的元素，并且固定定位不占用原来的位置
格式：position：fixed；
小技巧：使固定定位的元素贴着主板盒子。使用fixed和margin

定位的叠放顺序：z-index：数字--控制盒子的前后顺序，数越大越靠上

绝对定位盒子居中(绝对定位margin：0 auto方式失效)：首先left：50%，其次margin-left自身一半的宽度

元素显示与隐藏

  1. display:none–隐藏对象(不再占有原来的位置)
  2. display:block–除了转换为块元素以外，还有显示元素的意思
  3. visibility：visible–元素可视
  4. visibility：hidden–元素隐藏(保留原来的位置)
  5. overflow：visible–对溢出的内容不隐藏也不添加滚动条
  6. overflow：hidden–对溢出的内容隐藏
  7. overflow：scroll–不管超出内容与否，都添加滚动条
  8. overflow：auto–超出显示滚动条，不超出不显示

动画的使用

  1. 定义动画@keyframes+名字
  2. 调用动画animation-name:名字+animationduration:时间
  3. 动画序列：一个动画可以看做一段时间从0%持续到100%的过程，因此我们在定义动画内部，可以设置某个时间点的样式，比如：0%{transform:traslate(100px)} 50%{transform:traslate(0,100px}

动画常见属性

  1. @keyframes定义动画
  2. animation所有动画属性的简写，除了animation-play-state
  3. animation-name规定动画名称，必须的
  4. animation-duration规定动画完成一个周期花费的时间，默认是零，必须的
  5. animation-timing-fuction规定动画曲线，默认ease
  6. animation-delay规定动画何时开始，默认0
  7. animation-iteration-count规定动画播放次数，默认1，设置infinite可以变成无限播放
  8. animation-direction规定动画在下一周期逆向播放，默认是normal，alternate则是逆播放
  9. animation-play-state规定动画是否正在运行或者暂停，默认running，还有pause
  10. animation-fill-mode规定动画结束的状态，保持forwards，回到起始地点backwards

animation:动画名称 持续时间 动画曲线 何时开始播放次数 是否反方向 动画起始或者结束状态

  2. linear匀速
  3. ease-in低速开始
  4. ease-out低速结束
  5. ease-in-out低速开始且低速结束
  6. step()制定了时间函数的间隔数量

step如何理解呢：将一个过程分成n段。每次都是一瞬间到达下一段，而不是一个过程。比如：从左到右，8s内分8步，每s都会直接跳跃一步，而不是滑动过去的过程。这个属性值常常利用于展示一条长图，图里是每一帧的画面。

算术移位

算术移位是将二进制算数进行左移和右移的操作，看成十进制就是乘十和除十的操作，二进制也一样，左移就是相当于乘2，右移就相当于除2

原码的算数移位需要注意：符号位不变，左移需要在末尾补零，前端的舍弃。右移在前端补零，末尾舍弃。

反码的算术移位需要注意：如果是负数，那么反码的左右移需要补的就是1

补码的算术移位需要注意：如果是负数，那我们可以发现，从右边第一个1到最右边和原码相同，左边和反码相同，因此在算术移位时候，左移补0，右移补1



逻辑移位：可以理解为无符号数的算术移位

加减运算

原码的加减法对于计算机可能会出错，因此计算机一般都是用补码进行加减运算。
但是计算机在运算的时候，可能会出现溢出
溢出分为两种溢出

  1. 上溢–正数+正数=负数
  2. 下溢–负数+负数=正数

判断溢出：

  1. (加数的符号&&第二个加数的符号&&！结果的符号 )|| (！加数的符号&&！第二个加数的符号&&结果的符号)–如果结果为0则没有溢出，结果为1则溢出
  2. 根据符号位的进位和最高数值位的进位进行异或运算，结果位0则没有溢出，反之溢出
  3. 采用双符号位，如果结果的两个符号位的异或运算结果为0则没有溢出，反之也是溢出


符号拓展

c语言有一种功能是强制类型转换，这时候就需要用到符号拓展

拓展方式：

  1. 正数的原码反码补码拓展方式都是直接在前端加0，而小数是在末尾加0
  2. 负数的反码则是在符号位之前，最高数值位之后加1，小数则是在末尾加1
  3. 复数的补码根据前面补码与原码和反码的规律，我们可以知道整数是在符号之前，最高数值位之后加1，小数则是在末尾加0

浮点数

浮点数的表示格式：

  1. 浮点数由阶码和尾数构成
  2. 阶码一般由补码或移码构成的定点整数，尾数一般由补码或原码构成的定点小数
  3. 浮点数在理解上可以写成：2^阶码次方+尾数。而在这门课程中，我们一般写成：阶码；尾数
  4. 需要注意的是，阶码用逗号分开符号位和数值位，阶码和尾数用分号隔开
  5. 阶码的意义是表示尾数的小数点应该向前或者向后移动几位


浮点数尾数的规格化

  1. 左规–如果浮点数尾数最高位值数是无效数，那么就要进行左规
  2. 右规–如果浮点数尾数运算出现溢出(双符号位判断方式)，那么进行右规

左规的方式：
尾数算数位左移一位，阶码减1，直到尾数的最高位数值为有效值

右规的方式：
尾数算数位右移一位，阶码加1

cache-地址映射

全相联映射

  1. 将主存的地址分成(主存大小/Cache行长)这么多份
  2. 每一份都是一次性插入Cache的某一行中，并且记录插入的地址的二进制数位前(主存大小/Cache行长)位数

直接映射

  1. 依旧是将主存的地址分成(主存大小/Cache行长)这么多份
  2. 根据Cache的行数，将主存地址第(Cache行长前面Cache行数)那么多数位进行分配，每一位数都有对应可以放入Cache固定行
  3. 每一份都是一次性插入Cache的某一行中，并且记录插入的地址的二进制数位前(主存大小/(Cache行长+Cache行数))位数

组相联映射

  1. 将Cache分组(其实就是将Cache缩小了，比如原来8行，分组以后两行一组，可以看成四组)
  2. 复刻直接映射的步骤

格式：操作码+寻址特征+形式地址

系统总线：

系统总线是计算机内部系统各个部件相互连接的总线。可分为三类：数据总线，地址总线，控制总线

  1. 数据总线(双向：可以传输各功能部件的数据，包括指令操作数等等)
  2. 地址总线(单向：传输地址信息，比如主存地址和IO端口地址)
  3. 控制总线(可入可出：CPU送出控制命令，主存或者外设发出反馈信号)

  1. 单总线结构(cpu，主存，io设备都连接在一组总线上)

  3. 三总线结构(各部件之间采用三条总线构成通路，主存总线，io设备总线，DMA总线)

总线的性能指标

  5. 总线宽度(同时能传输的数据的位数，通常也叫数据总线的根数)
  6. 总线带宽(数据传输速率，总线带宽=总线宽度*工作频率)
  7. 总线复用(一种信号线可以在不同时间传输不同的信息)
  8. 信号线数(数据总线，地址总线，控制总线三种总线的总和)

奈氏准则

公式：理想低通信信道的极限传输速率=2W*log2(V)，单位是b/s;其中2W表示极限码元传输速率，W表示带宽，V是码元的种类

封装成帧

封装成帧是指在一段数据分别添加首部和尾部。首部和尾部有很多控制信息，包括接收方可以识别的帧的开始和结束

组帧的方法有四种

  1. 字符计数法(帧首部采用一个计数字符，来标明帧的字符数)
  2. 字节(符)填充法(发送方在原始数据内所有和首部，尾部，相同的字符前添加标识)
  3. 零比特填充法(“5110”法，在数据内只要有5个1连在一起，那么就在这5个1后加0)
  4. 违规编码法(比如在曼彻斯顿编码中，采用高-高和低-低作为帧的起始和终止)

停止-等待协议

顾名思义，发送方和接收方有停止和等待两步操作。每次发送完一个帧就停止发送，直到接收方发回确认信息才往下发送其他帧.如果迟迟没有等到确认信息，将再发送一次相同的帧

三种错误：

  1. 发送帧失败
  2. 确认信息发送失败(ACK丢失)
  3. 确认信息迟到(ACK迟到)

缺点：信道利用率极低

后退N帧协议(GBN)

发送方维持一组可以连续发送的帧的序号，每当前序的帧按顺序收到确认，那么序号向后移动，不然发送方会在计时器超时之后，继续发送相同的，未被确认的帧。
注意：如果发送方传输了0.1.2三帧，但接收方只返回了1的确认，那么0是默认传输成功了

  3. 接收端收到CTS以后开始发送数据帧(同时预约信道，告诉其他站点自己要用多久信道)
  4. 接收端接收到数据帧以后用CRC检验数据，没错则返回ACK
  5. 发送到收到ACK则发送下一数据帧，如果没收到则不断重复发送，直到发送到规定的最大发送次数

判断重传时机：

  1. 确定基本推迟时间：2*r
  2. 确定参数k，k=重传次数，k>10,那么不管重传次数多大，k永远=10
  3. 确立一个集合，这个集合包含的是0~(2^k)-1范围的所有整数
  4. 在集合随机选择一个数v，那么重传时间就是2rv
  5. 当重传次数达到16次还没有成功则放弃传输，并且向上级层报告

传输层概念

传输层两个协议：

  1. TCP协议(面向连接)-可靠，时延大，适用大文件
  2. UDP协议(无连接)-不可靠，时延小，适用小文件

传输层的寻址与端口：

  1. 复用：应用层所有进程都可以到传输层再到网络层

  2. UDP进行最大能力传输，但不保证可靠
  3. UDP面向报文，适合一次性传输少量数据的应用
  4. UDP无拥塞控制
  5. UDP首部开发小，仅仅8B，TCP则20B
  6. UDP根据应用层给多少报文就发多少

格式：

  1. 2B源端口号

  3. 2B的UDP长度

tcp协议特点和tcp报文段格式

  1. 面向连接，需要建立连接
  2. 每一条TCP只能由两个端点，是点对点
  3. TCP提供可靠传输服务
  4. TCP是双工通信(发送缓存-准备发送的数据和已发送的数据但没收到确认的数据。接收缓存-按序到达但是还没有被应用程序读取的数据和不按顺序到达数据)
  5. TCP面向字节流(TCP把应用程序送来的数据看成一段无结构字节流)

TCP报文首部格式：

  1. 序号：在TCP连接中传输的字节流中每一个字节都按照顺序编号，本字段则表示报文段要发送的数据的第一个字节的序号
  2. 确认号：期望收到对方下一个报文段第一个字节的序号。如果为N则说明N-1为止的数据都已经接收成功
  3. 数据偏移：TCP报文段数据部分距离起始位置有多远，以4B位位单位，一个数值是4B
  4. 紧急位URG：紧急位如果=1，则可以优先发送
  5. 确认位ACK：确认连接以后，ACK必须为1
  6. 推送位PSH：PSH=1则接收方尽快接受进程，不用等到缓存填满再向上交付
  7. 复位RST：RST=1则需要释放并且重新创建连接
  8. 同步为SYN：SYN=1说明这是请求建立连接报文
  9. 终止位FIN：FIN=1说明数据发送完毕，可以释放连接
  10. 窗口：允许对方发送的数据量
  11. 校验和
  12. 紧急指针：URG=1才有意义，指出本报中紧急数据文段字节数
  13. 选项
