操作系统的概念功能目标

操作系统概念：是指控制和管理整个计算机系统硬件和软件资源，并且合理的组织调度计算机的工作和资源分配，以提供给用户和其它软件方便的接口和环境。他是计算机系统中最基本的系统软件。(Windows任务管理器是最直观的例子)

操作系统作为资源管理者的功能：

  1. 处理机管理
  2. 存储器管理
  3. 文件管理
  4. 设备管理

打开QQ和别人视频的过程案例：

  1. 找到QQ安装的位置–文件管理
  2. 打开QQ.exe–存储器管理
  3. QQ程序正常运行–处理机管理
  4. 和朋友视频聊天，分配了摄像头–设备管理

操作系统作为用户和硬件之间的接口的功能：

  1. 联机命令接口(用户说一句，系统做一局)
  2. 脱机命令接口(用户说一堆，系统做一堆)
  3. 程序接口(允许用户通过程序间接使用)(由一组程序接口，即一组系统调用组成)

联机+脱机=命令接口(允许用户直接使用)

类比：操作系统就像是工人，硬件就像是锤子，锯子，钉子

操作系统的特征

  1. 并发(一件事情宏观上同时发生，微观上交替发生)
  2. 共享
    互斥共享：一个时间段只允许一个进程访问该资源
    同时共享：一个时间段宏观上允许同时进行访问资源，微观上是交替访问该资源
  3. 虚拟(物理上的实体变成若干个逻辑上的对应物)
    例子：打开了5个软件，在用户看起来是5个cpu同时为自己服务，其实是一个cpu交替处理各个服务进程
  4. 异步(允许多道程序并发执行，但由于资源有限，进程的执行不是一贯到底，而是走走停停)

操作系统的发展分类

  1. 手工操作
  2. 单道批处理(引入脱机输入输出技术，并监督程序负责控制作业输入输出)
  3. 多道批处理(多道程序并发执行)

分类：

  1. 分时操作系统
  2. 实时操作系统

操作系统的运行机制与体系结构

两种指令：

  1. 特权指令
  2. 非特权指令

两种处理器状态：

  1. 核心态
  2. 用户态

两种程序：

  1. 内核程序
  2. 应用程序

特权指令，核心态，内核程序不是普通用户可以执行的
非特权指令，用户态，应用程序是普通用户可以执行的

操作系统分层：

  1. 非内核功能
  2. 内核

内核：

  1. 进程管理，存储器管理，设备管理等
  2. 时钟管理，中断处理，原语(设备驱动，cpu切换等)

包含1和2功能叫做大内核
包含2功能叫做微内核

中断和异常

中断：

  1. 中断发生时，cpu立即变成核心态
  2. 中断发生后，当前运行的进程暂停，并由操作系统内核对中断进行处理
  3. 对于不同中断信号，有不同的处理

用户态->核心态只有中断方式实现
核心态->用户态只需要执行特权指令

中断分为内中断(指令终端，硬件故障，软件中断)和外中断(外设请求，人为干预)

系统调用

应用进程通过系统调用请求操作系统服务。由操作系统代为完成操作，更安全稳定。

按功能分类：

  1. 设备管理
  2. 文件管理
  3. 进程控制
  4. 进程通信
  5. 内存管理

系统调用的过程：

  1. 传递系统调用的参数
  2. 执行陷入指令(用户态)
  3. 执行系统调用相应的服务程序(核心态)
  4. 返回用户程序

进程的定义组成组织方式

程序：一个指令序列

进程：程序段，数据段，pcb组成了进程实体。创建进程就是创建pcb。撤销进程就是撤销pcb。pcb是进程存在的唯一标志

pcb的组成：

  1. 进程描述信息
  2. 进程控制和管理信息
  3. 资源分配清单
  4. 处理机相关信息

进程的组织方式：

  1. 链接方式：按照进程状态把pcb分为多个队列，操作系统持有各个队列指针
  2. 索引方式：根据进程状态建立索引表，操作系统拥有指向各个所以表的指针

进程的特征：

  1. 动态性
  2. 并发性
  3. 独立性
  4. 异步性
  5. 结构性

进程的状态与转换

三种基本状态：

  1. 运行态：占有cpu，并且在cpu上运行
  2. 就绪态：具备运行条件，由于没有空闲cpu，暂时不能运行
  3. 阻塞态：因等待某些事件而暂时不能运行

另外两种状态：

  1. 创建态：进程正在创建，操作系统分配资源，初始化pcb
  2. 终止态：进程从系统撤销，操作系统回收资源，撤销pcb

进程同步和进程互斥

同步亦称为直接制约关系，它是指为完成某种任务而建立多个进程，这些进程需要在某些位置上协调他们的工作次序而产生制约关系。如何解决异步问题，是进程同步讨论的内容

互斥亦称简介制约关系。进程互斥指当一个进程访问某临界资源时，另一个选购要访问临界资源的进程必须等待。当访问临界资源的进程访问结束，释放资源以后，另一个进程才能访问临界资源

对临界资源的互斥访问逻辑上分为四个部分：

  1. entry section进入区
  2. critical section临界区
  3. exit section退出区
  4. remainde sectionr剩余区

互斥访问遵循原则：

  1. 空闲让进
  2. 忙则等待
  3. 有限等待
  4. 让权等待

  1. 局部于管程的共享数据结构说明

管程的特征：

  3. 每次仅允许一个进程在管程内执行某个内部过程

具有快表的地址变换机构

时间局部性：如果执行了程序钟某条指令，那么不久后这条指令可能再次被执行。

空间局部性：一旦访问了某个存储单元，在不久之后，其附近的存储单元也可能被访问。

快表：又称联想寄存器，是一种访问速度比内存快多的高速缓冲寄存器，用于存放当前访问的若干页表项，以加速地址变换的过程，与之对应的内存被称为慢表

页面分配

驻留集：请求分页存储管理钟给进程分配的物理块集合

分配方式：

  1. 固定分配：分配固定数目物理块，驻留集大小不再改变
  2. 可变分配：分配一定数目物理块，根据情况增加减少，驻留集大小可以改变

置换策略：

  1. 局部置换：缺页时只能对自己的物理块进行置换
  2. 全局置换：可以把操作系统保留的空闲物理块分配给却也进程，也可以将其他进程持有的物理块置换到外存，在分配给缺页进程

I-O设备的概念和分类

什么是I-O设备：I-O就是输入和和输出，I-O设备就是可以将数据输入到计算机，或者接受计算机输出数据的外部设备，属于计算机中的硬件设备

I-O设备按使用特性分类：

  1. 人机交互类设备：鼠标键盘
  2. 存储设备：移动硬盘
  3. 网络通信设备：调制解调器

I-O设备按照传输速率分类：

  1. 低速设备：鼠标键盘
  2. 中速设备：打印机
  3. 高速设备：磁盘

I-O设备按照信息交换单位交换分类：

  1. 块设备：磁盘
  2. 字符设备：鼠标键盘

缓冲区：缓冲区是一个存储区域，可以由专门的硬件寄存器组成，也可以由内存组成

缓冲区的作用：

  1. 缓和COU和I/O设备之间的速度不匹配的矛盾

  4. 提高CPU和I/O设备之间并行性
