概念-组成-功能-分类

概念：计算机网络是指将分散的，具有独立性功能的计算机系统通过通信设备与线路连接起来，由功能完善的软件实现资源共享和信息传递的系统

计算机网络是互连(互联互通)的，自治(无主从关系)的计算机的集合

功能

1.数据通信–发邮件
2.资源共享–硬件(远程使用计算机),软件(某水果app)，数据(百度文档)
3.分布式处理–多台计算机共同处理工作任务的不同部分
4.提高可靠性–替代机
5.负载均衡

组成

1.硬件
2.软件
3.协议

工作方式

1.边缘部分：用户使用
2.核心部分：为边缘部分服务

功能组成

1.通信子网：实现数据通信
2.资源子网：资源共享，数据处理

按照视频的例子，资源子网就是对礼品的封装，处理，通信子网送礼物

1.2：城域网
1.3：局域网
1.4：个人区域网

2.按照使用者分

2.2：专用网–军用网

3.按照交换技术分
3.1：电路交换

4.1：总线型

速率相关的性能指标

速率

即数据率或者叫做数据的传输率或比特率
一般有几种单位：b/s，kb/s，mb/s(常说的兆)，gb/s，tb/s

带宽

最低频率与最高频率的差值，在计算机网络表示通信网络设备传输数据的能力(单位时间内从网络某一点到另一点通过的“最高数据率”)

吞吐量

单位时间内通过某个网络(信道，接口等)的数据量
–就好比我能吃100碗面，但我一口只能吃五碗

性能指标

时延

  1. 发送时延–数据从主机发送到信道的时间(数据长度/信道带宽(发送速率))
  2. 传播时延–数据在信道传播的时间(信道长度/电磁波在信道的传播速率)
  3. 排队时延–等待时间
  4. 处理时延–检测时间

时延带宽积

时延带宽积=时延(/s)*带宽(b/s)–数据量即某段信道链路有多少比特

往返时延RTT

RTT越大，在收到确认之前，可以传输的数据越多–好比反派在主角到来之前，可以做的坏事越多

利用率

  1. 信道利用率–有数据通过时间/总时间
  2. 网络利用率–信道利用率加权平均值

利用率越高，时延越高，可以想象为堵车

码元

码元是指用固定时长的信号波型(数字脉冲)，代表不同离散数值的基本波形

k进制码元可以想象成一栋楼有k层，而k进制码元可以携带log以2为底，以k为顶的那么多比特的数据量

码元传输速率和信息传输速率

单位是波特(Baud)，表示1s可以传输的码元

单位是比特，表示1s可以传输的数据量

奈氏准则

公式：理想低通信信道的极限传输速率=2W*log2(V)，单位是b/s;其中2W表示极限码元传输速率，W表示带宽，V是码元的种类

香农定理

公式：信道的极限传输速率=Wlog2(1+S/N)，单位是b/s;其中W是带宽，S(信道所传信号的平均功率)/N(高斯噪声的平均功率)是信噪比，信噪比又有另一种形式：信噪比(dB)=10*lg10(S/N)


基带信号和宽带信号

基带信号：将数字信号变成1和0两种电压，再送到数字信道(基带传输)

宽带信号：将基带信号进行调制以后，送到模拟信道(宽带传输)

模拟数据->PCM编码器->数字信号

变成数字信号就是编制
变成模拟信号就是调制

封装成帧

封装成帧是指在一段数据分别添加首部和尾部。首部和尾部有很多控制信息，包括接收方可以识别的帧的开始和结束

组帧的方法有四种

  1. 字符计数法(帧首部采用一个计数字符，来标明帧的字符数)
  2. 字节(符)填充法(发送方在原始数据内所有和首部，尾部，相同的字符前添加标识)
  3. 零比特填充法(“5110”法，在数据内只要有5个1连在一起，那么就在这5个1后加0)
  4. 违规编码法(比如在曼彻斯顿编码中，采用高-高和低-低作为帧的起始和终止)

纠错

奇偶校验码：分奇校验和偶校验，即查看一段数据中1的个数是奇数还是偶数。

CRC循环冗余码：

  1. 发送端：要传的数据+(要传的数据%生成多项式)。在模之前，需要根据生成多项式的最高阶数给要传的数据后面添加0
  2. 接收端：接收的数据%生成多项式。如果结果为0则没有出错

海明码：太难，我也看不懂

数据链路层的流量控制

  1. 停止-等待协议
  2. 滑动窗口协议(后退N帧协议，选择重传协议)

注意：滑动窗口长度W最好>=1&&<=(2^n)-1,其中n是用比特对帧编号

停止-等待协议

顾名思义，发送方和接收方有停止和等待两步操作。每次发送完一个帧就停止发送，直到接收方发回确认信息才往下发送其他帧.如果迟迟没有等到确认信息，将再发送一次相同的帧

三种错误：

  1. 发送帧失败
  2. 确认信息发送失败(ACK丢失)
  3. 确认信息迟到(ACK迟到)

缺点：信道利用率极低

后退N帧协议(GBN)

发送方维持一组可以连续发送的帧的序号，每当前序的帧按顺序收到确认，那么序号向后移动，不然发送方会在计时器超时之后，继续发送相同的，未被确认的帧。
注意：如果发送方传输了0.1.2三帧，但接收方只返回了1的确认，那么0是默认传输成功了

csma-ca协议

CSMA-CA工作步骤

  1. 发射出RTS(发射端地址，接收端地址，发送时间等)
  2. 接收端收到RTS之后，返回CTS
  3. 接收端收到CTS以后开始发送数据帧(同时预约信道，告诉其他站点自己要用多久信道)
  4. 接收端接收到数据帧以后用CRC检验数据，没错则返回ACK
  5. 发送到收到ACK则发送下一数据帧，如果没收到则不断重复发送，直到发送到规定的最大发送次数

csma-cd协议

CS: 监听：检查信道有没有其他计算机发送数据
MA：多点接入：许多计算机以多点接入方式连接在一根总线上
CD：碰撞检测：监听电压情况，判断自己在传输数据的时候其他站有没有传输数据

判断重传时机：

  1. 确定基本推迟时间：2*r
  2. 确定参数k，k=重传次数，k>10,那么不管重传次数多大，k永远=10
  3. 确立一个集合，这个集合包含的是0~(2^k)-1范围的所有整数
  4. 在集合随机选择一个数v，那么重传时间就是2rv
  5. 当重传次数达到16次还没有成功则放弃传输，并且向上级层报告

最小帧长：帧长(bit)/数据传输速率>=2r。所以最小帧长=2r数据传输速率

电路交换

特点：独占资源

步骤：

  1. 建立连接(呼叫)
  2. 通信
  3. 释放连接(释放电路)

优点：

  1. 通信时延小
  2. 有序传输
  3. 没有冲突
  4. 实时性强

缺点：

  1. 建立连接时间长
  2. 线路独占，使用效率低
  3. 灵活性差
  4. 没有纠错控制能力

报文交换

特点：发送源应用发送的信息整体

步骤：

  1. 传递到交换设备
  2. 交换设备进行存储转发
  3. 根据网络情况选择线路

优点：

  1. 无需建立连接
  2. 存储转发，动态分配线路
  3. 线路可靠性高
  4. 线路利用率高
  5. 多目标服务

缺点：

  1. 存储转发有时延
  2. 需要网络节点有较大的缓存存储的空间

分组交换

特点：把大的数据报分成若干个小的数据块

步骤：

  1. 分割数据报
  2. 将第一个数据报发送给交换设备，由交换设备选择路径传输
  3. 过一段时间继续发送第二个数据报，以此类推

优点：

  1. 无需建立连接
  2. 存储转发，动态分配线路
  3. 线路可靠性高
  4. 线路利用率高
  5. 存储管理更容易

缺点：

  1. 有存储转发时延
  2. 需要传输额外的信息量
  3. 乱序到目的主机，需要重新排序

数据报与虚电路

虚电路：虚电路将数据报和电路方式结合，发挥二者优点

步骤：

  1. 源主机发送“呼叫请求”分组，并收到“呼叫答应”分组，源主机和目标主机建立连接
  2. 每个分组携带“虚电路号”，由路由器识别，可以让每个分组按照固定的道路传输到目的主机

传输层概念

  1. 为应用层提供通信服务
  2. 使用网络层提供的服务

功能：

  1. 提供进程与进程的逻辑通信
  2. 复用(把所有信件塞到信箱)和分用(快递员分发信件)

传输层两个协议：

  1. TCP协议(面向连接)-可靠，时延大，适用大文件
  2. UDP协议(无连接)-不可靠，时延小，适用小文件

传输层的寻址与端口：

  1. 复用：应用层所有进程都可以到传输层再到网络层
  2. 分用：传输层从网络层收到数据后交付指明的应用进程

端口：传输层的SAP，标识主机的应用进程。端口只有本地意义，在不同计算机相同端口没有联系

udp协议

  1. UDP无连接，减少时延
  2. UDP进行最大能力传输，但不保证可靠
  3. UDP面向报文，适合一次性传输少量数据的应用
  4. UDP无拥塞控制
  5. UDP首部开发小，仅仅8B，TCP则20B
  6. UDP根据应用层给多少报文就发多少

格式：

  1. 2B源端口号
  2. 2B目的端口号
  3. 2B的UDP长度
  4. 2B的UDP校验和
  5. 数据

找不到目的端口号或者根据校验和校验出错则丢弃

tcp协议特点和tcp报文段格式

  1. 面向连接，需要建立连接
  2. 每一条TCP只能由两个端点，是点对点
  3. TCP提供可靠传输服务
  4. TCP是双工通信(发送缓存-准备发送的数据和已发送的数据但没收到确认的数据。接收缓存-按序到达但是还没有被应用程序读取的数据和不按顺序到达数据)
  5. TCP面向字节流(TCP把应用程序送来的数据看成一段无结构字节流)

TCP报文首部格式：

  1. 序号：在TCP连接中传输的字节流中每一个字节都按照顺序编号，本字段则表示报文段要发送的数据的第一个字节的序号
  2. 确认号：期望收到对方下一个报文段第一个字节的序号。如果为N则说明N-1为止的数据都已经接收成功
  3. 数据偏移：TCP报文段数据部分距离起始位置有多远，以4B位位单位，一个数值是4B
  4. 紧急位URG：紧急位如果=1，则可以优先发送
  5. 确认位ACK：确认连接以后，ACK必须为1
  6. 推送位PSH：PSH=1则接收方尽快接受进程，不用等到缓存填满再向上交付
  7. 复位RST：RST=1则需要释放并且重新创建连接
  8. 同步为SYN：SYN=1说明这是请求建立连接报文
  9. 终止位FIN：FIN=1说明数据发送完毕，可以释放连接
  10. 窗口：允许对方发送的数据量
  11. 校验和
  12. 紧急指针：URG=1才有意义，指出本报中紧急数据文段字节数
  13. 选项

网络应用模型

应用层概述：应用层对应用程序提供通信服务

应用层的功能：

  1. 文件传输，访问，管理
  2. 电子邮件
  3. 虚拟终端
  4. 查询系统和远程作业登录

应用层的重要协议：

  1. FTP
  2. SMTP,POP3
  3. HTTP
  4. DNS

网络应用模型：

  1. 客户/服务器模型
  2. P2P模型

C/S(客户/服务器模型)：

  1. 服务器：提供计算服务的设备，永久提供服务，永久性访问地址/域名
  2. 客户机：请求计算服务的主机，与服务器通信，使用服务器提供的服务，间接性接入网络，可能是使用动态IP地址，不与其他客户机直接通信

P2P模型：

  1. 每个主机都可以提供服务，都可以请求服务
  2. 不存在永久在线的服务器
  3. 任意端系统，节点可以直接通信
  4. 节点可以间接性接入网络，可以改变IP地址
  5. 可拓展性好
  6. 网络健壮性强
